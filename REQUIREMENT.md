# World Road Trip 지도 웹페이지 — 요구사항 정리

> 저장소 이름을 ItalyRoadTrip → **WorldRoadTrip**으로 변경 (`https://github.com/netscomm/WorldRoadTrip`), 이탈리아 외 다른 여행도 같은 구조로 다룰 수 있도록 확장 예정.

## 데이터 소스

- 가민 FIT 파일(`F:\DCIM\FIT files`): 이탈리아 자전거 여행 경로 정보.
- DJI 오즈모 mp4 영상(`F:\DCIM\DJI_001`, 약 200GB): 한국시간대로 기록되어 있어 이탈리아 시간으로 일괄 보정 필요.
- (추후 추가 예정) 갤럭시 폰 사진, 자전거 여행 중 작가가 찍어준 사진.

## 지도/마커 기능

- 지도에 내가 다녀간 경로(FIT 트랙)를 표시.
- 촬영이 시작된 지점에 마커 표시.
  - 이동 중 촬영한 영상(A→B 이동 중 촬영)은 **촬영 시작 지점(A)**에 마커.
  - 멈춰서 촬영한 경우도 동일하게 마커로 재생 가능해야 함.
- 오즈모 영상은 GPS가 없으므로 FIT 정보로 위치를 추측, GPS 메타데이터가 있는 미디어는 해당 GPS 사용.
- FIT 파일별로 트랙 색상을 다르게 표시(겹치는 구간 구분), 마커 색상도 해당 트랙 색과 맞춰서 구분.

## 마커 상호작용

- 마커에 마우스를 hover하면 동영상/사진 재생.
- 클릭하면 hover 없이도 재생 유지(잠금).
- 마우스를 올렸을 때 **위도, 경도, 날짜, 시간**도 같이 표시.
- 동영상의 **시작 시간, 종료 시간**도 표시.
- 이 정보들을 **복사·붙여넣기 가능**하게 (시간 추정에 활용하기 위해).
- 동영상 클릭 시 재생되는 파일의 **절대경로**를 알 수 있어야 함(경로 불일치 확인용).
- 영상 재생 영역이 전체 페이지의 **1/4 정도 크기**를 차지하도록.

## 경사 구분

- 동영상의 시작/끝 위치(고도)를 비교해서 내리막/오르막/평지를 구분하고, 마커에 표시.
- 미분류(경사 "알 수 없음")로 분류된 영상들을 한눈에 볼 수 있게 목록으로 제공 — 잘못 미분류된 게 있는지 확인하기 위함.

## 범례/탐색

- FIT 파일별 범주(범례)를 지도 위에 표기.
- 범례 클릭 시 해당 FIT 트랙 위치로 지도 이동.

## 기타 운영 관련

- 결과물(완료 내용, 알아두실 점)을 md 파일로 따로 보관, "1st Release"임을 명시.
- PC 재부팅/다른 PC에서 실행할 때 사전 준비사항도 md에 명시.
- 요구사항 자체도 별도 md 파일(`REQUIREMENT.md`)로 정리.

## 데이터 보정 (대화 중 함께 도출됨)

- FIT timestamp: UTC 기록 → +2시간(CEST) 보정.
- DJI 영상: 2026-06-10 00:00 이전 클립은 -7시간(한국→이탈리아), 이후 클립은 -1시간(카메라 시계가 현지시간과 비슷하게 바뀌었지만 1시간 빠르게 기록된 것으로 확인) — 사용자가 알려준 실제 위치/시간으로 검증하며 확정.

## GitHub 공개

- 개인 GitHub 저장소(`netscomm/WorldRoadTrip`)에 프로젝트 push.
- `index.html` 웹페이지는 다른 사람도 볼 수 있게 GitHub Pages로 공개.
- 단, 로컬 영상 파일(F:\DCIM\DJI_001, ~200GB)은 GitHub에 올리지 않으며, `file://` 경로 특성상 영상 재생은 **본인 PC에서만** 가능 — 무료 GitHub 계정 제약상 Pages를 켜려면 저장소를 public으로 만들어야 하고, 이에 따라 `data.js`에 포함된 로컬 경로/PC 사용자명이 공개되는 것은 감수하기로 함.
